# SQL Data Analysis Portfolio

**Building my SQL skills one query at a time** — starting from the LinkedIn “SQL for Data Analysis” course.

### About This Project
This repository is my personal learning playground.  
I started with zero local SQL setup and slowly built 25+ queries on the Superstore Sales dataset and then added a second dataset (Returns) to practice JOINs.  
Everything is now organized into clean, focused `.sql` files so anyone (including future recruiters) can easily see my thinking process.

### Organized Query Files
- **`01_basic_exploration.sql`** – Basic SELECT and first look at the data  
- **`02_aggregations.sql`** – GROUP BY, SUM, AVG, COUNT, etc.  
- **`03_time_analysis.sql`** – Date functions, yearly and monthly trends  
- **`04_advanced_filters.sql`** – HAVING, WHERE + GROUP BY, COUNT(DISTINCT), calculated columns  
- **`05_joins.sql`** – LEFT JOINs with the Returns dataset (9 real JOIN queries)

### Skills Demonstrated
- SELECT, LIMIT, ORDER BY  
- GROUP BY + aggregate functions (SUM, AVG, COUNT, COUNT(DISTINCT))  
- Date functions (strftime)  
- Filtering with WHERE and HAVING  
- Calculated columns and profit margin calculations  
- LEFT JOINs across two tables  
- Return rate calculations using CASE  

### What I Learned
This project taught me how to go from copying queries to owning every line. I learned the importance of clean organization, commenting my work, and how small repetitive practice turns “foreign vibes” into real familiarity. I still have a lot to learn (subqueries and window functions are next), but I’m proud of the steady progress I've made from zero.

### How to Run the Queries
1. Download the two CSV files from the repo (`superstore_sales.csv` and `superstore_returns.csv`).  
2. Open them in any SQL tool that supports CSV import (BigQuery, SQLite, DBeaver, etc.).  
3. Run the queries in any of the `.sql` files — they are all self-contained.
